**P0**: 該当なし

**[P1-1] フェーズ 0 のベースライン計測とフェーズ 1 の依存関係の明確化**
- **対象セクション**: 9. 実装・リリース計画 フェーズ 0 / フェーズ 1
- **内容**: フェーズ 0 でベースライン計測を実施し、その結果をフェーズ 1 のステージ別 delta 初期値に反映するとあるが、フェーズ 0 の計測結果をどの時点で RFC に反映するかが不明確である。フェーズ 0 完了後に RFC を更新してからフェーズ 1 に進むのか、フェーズ 0 の結果を実装時に直接反映するのかで、レビュー可能性が異なる。
- **修正の期待値**: フェーズ 0 完了後に計測結果を Section 5.1.4 に追記し、delta 初期値を確定させてからフェーズ 1 に進む旨を明記する。

**[P1-2] MPC との相互作用に関する仮説の検証失敗時の対応**
- **対象セクション**: 7.1 スケーラビリティとパフォーマンス
- **内容**: MPC が評価値を安定化させるという仮説を述べた上で「両条件下でベンチマークを実施し、この仮説を検証する」とあるが、仮説が棄却された場合（MPC ON 時に retry が増加した場合）の対応方針が述べられていない。
- **修正の期待値**: MPC ON 時に retry 発生率が悪化した場合のアクション（例: MPC ON 専用の delta テーブルを検討する、あるいは MPC ON 時は Aspiration チューニングを無効化する等）を撤退基準に追加する。
